## Executive Summary

This security audit of the YouBai Anti-Counterfeiting Traceability System identified **34 security findings** across the application. Of these, **31 are confirmed** with closed evidence chains, **3 are highly likely** and await additional verification, and **0 require further evidence or were rejected**.

The system is a monolithic PHP application using the deprecated `mysql_*` extension with direct SQL string concatenation throughout. It has **no parameterized queries, no input sanitization, no CSRF protection, no security headers, and no audit logging**. The application relies on unsalted MD5 for password storage and uses a flat role-based access model with no per-resource scope boundaries.

**Critical risk highlights:**
- **Authentication bypass via SQL injection** is possible on all three login paths (admin, agent, consumer query)
- **Unauthenticated SQL injection** exists on the consumer query page and barcode preview page — the public-facing trust mechanism is directly attackable
- **The installer is publicly accessible** when `install.lock` is absent (not included in distribution), enabling complete system takeover including arbitrary code execution
- **Bulk code operations** (enable/disable, delete) lack ownership scope, allowing any authenticated admin to manipulate all anti-counterfeiting codes globally
- **Agent PII** (phone, WeChat, QQ, address) is exposed to unauthenticated consumers on every product query

The absence of foundational security controls creates systemic risk: every SQL query in the application is a potential injection point, every state-changing form is a CSRF target, and every admin action is unauditable.

## Confirmed Finding Details

### RC-001-REV-001 — SQL Injection in Admin Login

| Field | Detail |
|---|---|
| **ID** | RC-001-REV-001 |
| **Severity** | Critical |
| **Bound Function** | BF-006 System Configuration / SF-006 Admin Authentication |
| **Entry Point** | `manage/index.php?act=adminlogin` (HTTP POST) |
| **Sensitive Point** | `mysql_query` at `manage/index.php:51` |
| **Control Failure** | No input sanitization — `$username` from `$_POST["Username"]` directly concatenated into SQL at line 50 |

**Evidence & Data Flow:**
1. Source: `$username = trim($_POST["Username"])` — `manage/index.php:46`
2. Transit: `$sql = "select * from tgs_admin where username='".$username."' and password='".md5($password)."'"` — `manage/index.php:50`
3. Sink: `$res = mysql_query($sql)` — `manage/index.php:51`

**Why It Holds:** Attacker can inject `' OR '1'='1` or `' UNION SELECT` via the username field to bypass password check or extract all admin credentials. No `mysql_real_escape_string`, no prepared statement, no parameterized query.

**Business Impact:** Full admin account compromise. Attacker gains superadmin access to all anti-counterfeiting codes, agent data, product data, and system configuration.

**Remediation:** Replace direct string concatenation with prepared statements (PDO/mysqli). Apply input validation. Migrate from deprecated `mysql_*` to PDO.

### RC-001-REV-012 — MD5 Password Hashing Without Salt

| Field | Detail |
|---|---|
| **ID** | RC-001-REV-012 |
| **Severity** | High |
| **Bound Function** | BF-006 System Configuration / SF-006 Admin Authentication, SF-011 Agent Authentication, SF-012 Agent Profile |
| **Entry Points** | `manage/index.php:50`, `agent/login.php:84`, `agent/edit.php:205` |
| **Control Failure** | `md5()` without salt; no bcrypt/argon2 |

**Evidence:** Password hashed via `md5()` at `manage/index.php:50`, `agent/login.php:84`. Stored in `tgs_admin` and `tgs_agent` tables. Additionally, DB credentials hardcoded in `data/conn.php` with empty default `$sys_key`.

**Why It Holds:** MD5 is cryptographically broken. Combined with SQL injection enabling database exfiltration, all passwords are effectively plaintext.

**Business Impact:** Once database is compromised (via any SQLi), all admin and agent credentials are recoverable via rainbow tables.

**Remediation:** Migrate to `password_hash()` with `PASSWORD_BCRYPT` or `PASSWORD_ARGON2ID`. Rehash existing passwords on next login.

### RC-002-REV-001 — Bulk Code Status Change Without Ownership or Scope Boundary

| Field | Detail |
|---|---|
| **ID** | RC-002-REV-001 |
| **Severity** | Critical |
| **Bound Function** | BF-001 Anti-Counterfeiting Code Management / SF-003 Code Status Management |
| **Entry Point** | `manage/zt_fwm.php` (HTTP POST, various `act` parameters) |
| **Sensitive Point** | Bulk `UPDATE tgs_code SET qiyong` at `manage/zt_fwm.php:680-750` |
| **Control Failure** | No ownership check; no product/agent scope boundary; no volume limit; no audit log; no glqx level restriction |

**Evidence & Data Flow:**
